# Open Questions

This document tracks unresolved design questions.

These should not be prematurely decided.

---

## 1. Final Module Schema

The final schema is intentionally undecided.

The schema will be refined only after multiple modules have been implemented.

---

## 2. Curriculum Source

Possible starting standards include:

- US Common Core
- UK National Curriculum
- International Baccalaureate
- Other national standards

The project may eventually support multiple standards.

---

## 3. Module File Format

Possible formats:

- Markdown
- YAML
- JSON
- Markdown + frontmatter

The format should remain:

- Git-friendly
- human-readable
- machine-readable

---

## 4. Licensing Strategy

Questions include:

- preferred use of OER materials
- public domain textbooks
- permissive licenses
- rewritten content vs quoted content

Traceability must remain mandatory.

---

## 5. Module Granularity

Even when using curriculum standards, the ideal module size may require adjustment.

Observations from real module creation will guide refinement.

---

## 6. Testing and Verification

Potential future module components:

- mastery tests
- diagnostic questions
- prerequisite validation

These may belong outside the core module definition.

---

## 7. Multi-language Support

The repository may eventually support multiple languages.

It is unclear whether modules should be:

- language-neutral with translations
or
- language-specific modules
